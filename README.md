FEE-247 (DEVELOPMENT OF IOT SOLUTIONS WITH LORA AND LORAWAN)

This Repository is an IoT Solution using Three-Phase Methodology for a LoRaWAN End Device ABP connecting on TTN over a 
Single Channel LoRaWAN Gateway, by MoT protocol using a Device PKLoRa-WAN composed in base of ESP32 core & LoRa radio RFM95
The End Device LoRaWAN is composed in addtional of Sensors LDR & DHT22 & GPS & Oled 0.96

This repository is composed of 6 Levels of an IoT Solution developed through Three-Phase Methodology.

Level 1 - The LoRaWAN End Device ABP collects data from the sensors LDR & DHT22 & GPS and transmits this data via LoRa radio
          using the MoT protocol. This device is declared in a TTN application.


Level 2 - The LoRaWAN Single Channel Gateway, also declared in the TTN gateways, receives this data, and transmits them
          through Wi-Fi Internet connection to TTN Application Server.


Level 3 - The Edge Nivel_3 in Python, do a connection to TTN Application Server, and collects the data from the Application
          using an API Key login, via MQTT (TLS port 8883) and save the data from the LDR + DHT22 + GPS sensors in 
          the Level 4 - Nivel4.csv file.

Level 4 - Nivel4.csv file - function to save the data collected from the LDR + DHT22 + GPS sensors, as application data
          As management data, uplinks rssi & snr, gateway_id, End Device_ID, battery level is collected.


Level 5 - Do the data abstraction in Python - Reads in real time the file from Level 4 generated by Level 3 (collects TTN via MQTT):
          # timestamp,uplink_counter,ldr_value,intensity,
          # rssi,snr,gateway_id,MY_ID,battery,temperature,
          # humidity,latitude,longitude
          # Calculates cumulative averages and saves ONCE per cycle:
          #
          # - medidas_aplicacao_media.txt → read by N6 (Level 6)
          # - medidas_gerencia_media.txt → read by N6 (Level 6)
          #
          # Average RSSI (dBm) and SNR (dB) via linear domain (mW):
          # - RSSI: converts dBm → mW, accumulates, calculates average, converts back to dBm
          # - SNR: converts dB → linear ratio, accumulates, calculates average, converts back to dB
          # This avoids the distortion that the direct arithmetic average in dB produces.
